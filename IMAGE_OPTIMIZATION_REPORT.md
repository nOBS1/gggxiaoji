## 📊 优化概览

### 问题描述
项目中的鸡蛋图片资源体积过大，影响加载性能：
- 6个鸡蛋PNG图片总大小：**4.84 MB**
- 单个图片大小：**700-900 KB**
- 分辨率：**1024x1024 像素**

### 优化策略
1. **降低分辨率**：1024x1024 → 256x256 (Web显示足够)
2. **PNG压缩**：使用高质量压缩（质量85%）
3. **保留原图**：原图备份到 `public/eggs_backup/`

## 📝 技术细节

### PNG 压缩原理
1. **缩小分辨率**：使用高质量缩略图算法
2. **PNG编码**：使用 .NET 的 PNG 编码器
3. **质量参数**：设置为 85%（平衡质量和体积）

### 代码实现
```powershell
# 加载原图
$img = [System.Drawing.Image]::FromFile($file.FullName)

# 创建256x256缩略图
$thumbnail = $img.GetThumbnailImage(256, 256, $null, [IntPtr]::Zero)

# 使用PNG编码器保存
$pngEncoder = [System.Drawing.Imaging.ImageCodecInfo]::GetImageEncoders() | 
    Where-Object { $_.MimeType -eq "image/png" }
$thumbnail.Save($outputPath, $pngEncoder, $encoderParams)
```

## 🚀 进一步优化建议

### 1. 转换为 WebP 格式（可选）
WebP 格式可以进一步减少 25-35% 的体积：

| 格式 | 当前大小 | WebP预期 | 额外减少 |
|------|----------|----------|----------|
| PNG (256x256) | 367 KB | ~240-280 KB | ~25-35% |

**实现方式**：
- 使用 `cwebp` 命令行工具
- 或使用在线服务：https://squoosh.app

**注意**：需要更新 `config.js` 中的图片路径扩展名。

### 2. 使用 Vite 图片优化插件
在构建时自动优化图片：
```javascript
// vite.config.js
import { imagetools } from 'vite-imagetools'

export default {
  plugins: [imagetools()]
}
```

### 3. 启用 Gzip/Brotli 压缩
在生产服务器上启用压缩，进一步减少传输大小：
- Gzip：约减少 10-20%
- Brotli：约减少 15-25%
