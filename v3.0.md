# 小趣闻·啄米鸡 v3.0 版本总结

## 项目概述

**小趣闻·啄米鸡** 是一款基于 Web 的放置类点击游戏，玩家通过点击屏幕让小鸡啄米，收集各种稀有度的蛋，并在市场上进行交易。项目采用前后端分离架构，支持本地试玩和在线同步两种模式。

---

## 技术栈

### 前端技术

- **核心语言**: 原生 JavaScript (ES6+)
- **界面**: HTML5 + CSS3
- **模块化**: ES Modules
- **国际化**: 自定义 i18n 系统（中英双语）
- **构建工具**: Vite
- **数据分析**: Google Analytics (gtag.js)
- **代码质量**: ESLint

### 后端技术

- **运行时**: Cloudflare Workers (Serverless)
- **Web 框架**: Hono (轻量级 TypeScript 框架)
- **开发语言**: TypeScript
- **数据库**: Supabase (PostgreSQL)
- **认证系统**: 
  - JWT (JSON Web Tokens)
  - Google OAuth 2.0 第三方登录
- **部署工具**: Wrangler (Cloudflare Workers CLI)

### 数据存储

#### 远程数据库 (Supabase PostgreSQL)
- 用户信息和档案
- 游戏状态和进度
- 升级和任务数据
- 市场订单和交易记录
- 金币变动历史

#### 本地存储 (LocalStorage)
- 游戏状态缓存（离线试玩）
- 用户偏好设置（语言、音效等）
- 金币变动日志（前端查询优化）

## 核心功能

### 游戏机制

1. **点击系统**
   - 基础点击进度累积
   - 升级提升点击力量
   - 保底机制保证掉落

2. **掉落系统**
   - 多种稀有度蛋类：白、棕、银、金、紫、黑、金色
   - 幸运值影响稀有掉落概率
   - 自动出售系统（可升级解锁）

3. **升级系统**
   - 钻石升级：点击力量、幸运加成、自动出售、金币加成
   - 金币升级：增加稀有掉落率、自动出售低级蛋、提升金币收益

4. **任务系统**
   - 每日点击任务
   - 收集特定稀有度蛋任务
   - 市场交易任务
   - 自动重置和奖励发放

5. **商店系统**
   - 蛋类出售（排除紫、黑、金色稀有蛋）
   - 钻石购买（观看广告获得）
   - 被动收入机制

### 市场交易系统 (v2.2 新增)

1. **订单管理**
   - 创建卖单（设定价格和数量）
   - 购买其他玩家订单
   - 取消自己的订单
   - 交易手续费机制

2. **市场界面**
   - 按稀有度筛选
   - 价格区间筛选
   - 关键词搜索
   - 排序功能（价格、时间、单价）
   - 分页显示（支持自定义每页数量）
   - 自动刷新（30秒间隔）

3. **统计功能**
   - 市场总订单数
   - 各稀有度蛋挂单统计
   - 24小时交易量
   - 最高/最低价格

4. **用户体验优化**
   - 详细错误提示（显示当前/需求数量）
   - 我的订单独立标签页
   - 快速价格筛选按钮
   - 响应式设计

### 金币记录系统 (v3.0 新增)

1. **金币变动日志**
   - 记录所有金币收入和支出
   - 包含时间戳、类型、描述、变动后余额
   - 本地存储最近 100 条记录

2. **记录来源**
   - 市场购买/出售
   - 蛋类出售
   - 任务奖励
   - 广告奖励
   - 被动收入
   - 升级消费

3. **查询界面**
   - 按类型筛选（收入/支出/全部）
   - 时间倒序排列
   - 格式化显示金额和时间
   - 实时更新

### 认证系统

1. **邮箱注册/登录**
   - 密码强度检测
   - 昵称设置
   - JWT Token 认证

2. **Google OAuth 登录**
   - 一键授权登录
   - 自动创建用户档案
   - 安全令牌传递

3. **数据同步**
   - 登录后自动上传本地数据
   - 服务器数据覆盖本地
   - 防止数据冲突

## 架构特点

### 前后端分离

- **前端**: 静态资源部署（Cloudflare Pages / 静态托管）
- **后端**: Serverless API（Cloudflare Workers）
- **优势**: 
  - 高可用性和扩展性
  - 低运维成本
  - 全球 CDN 加速

### 渐进式增强

- **离线试玩模式**: 无需登录，数据存储在本地
- **在线同步模式**: 登录后数据存储在云端
- **无缝切换**: 本地数据可自动上传云端

### 安全与隐私

1. **AdSense 合规**
   - 后端响应不返回用户 ID 和邮箱
   - 仅返回昵称等非敏感信息
   - JWT Token 使用 HttpOnly Cookie（推荐）

2. **数据验证**
   - 后端业务逻辑验证（防止作弊）
   - 前端仅作为缓存和 UI 层
   - RPC 函数原子操作保证数据一致性

3. **跨域安全**
   - CORS 配置仅允许授权域名
   - 环境变量管理敏感配置
   - OAuth 回调 URI 白名单

## 部署流程

### 前端部署

```bash
cd H:\cs\xiaoji-game
npm run build
# 将 dist 目录部署到 Cloudflare Pages 或其他静态托管
```

### 后端部署

```bash
cd H:\cs\xiaoji-game\api
wrangler deploy
# 自动部署到 Cloudflare Workers
```

### 环境配置

1. **Cloudflare Workers 环境变量**:
   - `SUPABASE_URL`
   - `SUPABASE_SERVICE_KEY`
   - `JWT_SECRET`
   - `GOOGLE_CLIENT_ID`
   - `GOOGLE_CLIENT_SECRET`
   - `GOOGLE_REDIRECT_URI`
   - `CORS_ORIGIN`

2. **本地开发环境变量** (`.dev.vars`):
   - 包含所有敏感配置
   - 不提交到版本控制

## 数据库架构

### 主要数据表

1. **users** - 用户账户
2. **profiles** - 用户档案（昵称、创建时间）
3. **game_state** - 游戏状态（金币、钻石、进度）
4. **upgrades** - 用户升级等级
5. **inventory** - 用户库存（蛋类数量）
6. **tasks** - 任务进度
7. **game_stats** - 游戏统计
8. **market_orders** - 市场订单
9. **market_transactions** - 交易历史

### RPC 函数

- `create_market_order` - 创建卖单
- `buy_market_order` - 购买订单
- `cancel_market_order` - 取消订单
- `get_market_stats` - 获取市场统计
